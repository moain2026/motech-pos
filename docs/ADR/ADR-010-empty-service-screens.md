# ADR-010 — شاشات خدمية/مكررة بلا وظيفة مستقلة (POST_WST / POSADVS2)

- **الحالة:** مقبول (📄 N/A — مغلقة معمارياً)
- **التاريخ:** 2026-07-06
- **الشاشات:** POST_WST، POSADVS2

## السياق

### POST_WST (10.5 KB — شبه فارغة)
- **proof:** `docs/screens/POST_WST.md` → بلوكان فقط (`BLOCK8` تحكّم + `MTX_WST_TRANS` DB)، **canvas واحد**، لا ملف مساعدة. `MTX_WST_TRANS` جدول مناولة/هدر مستودعي (WMS-adjacent). أصغر شاشة POST وأفقرها بنيةً — قالب خدمي بلا منطق بيع مستقل.

### POSADVS2 (105 KB — نسخة لمسية مكرّرة)
- **proof:** `docs/screens/POSADVS2.md §1` → فئة «شاشات لمس/متقدمة»، جداولها `IAS_POS_HUNG_BILLS`, `IAS_POS_MACHINE`, `IAS_PARA_GEN` — نفس نطاق شاشة البيع اللمسية POSADVS. متغيّر ثانٍ من الواجهة اللمسية (skin/تخطيط بديل) لا وظيفة جديدة.

## القرار

**لا تُبنى** أيٌّ منهما كشاشة مستقلة.
- POST_WST: N/A — لا وظيفة تشغيلية فعلية (وما يتصل بالمناولة المستودعية يقع تحت ADR-006 WMS).
- POSADVS2: N/A — تغطيها `PosPage` الحديثة (شاشة بيع لمسية واحدة متجاوبة تُغني عن تعدد نسخ الـForms).

## البديل

- **البيع اللمسي** كله في `PosPage` (features/pos-terminal): بحث/باركود/شبكة أصناف لمسية + سلة + دفع متعدد + تعليق/استئناف — واجهة واحدة متجاوبة (touch-first) بدل POSADVS/POSADVS2 المنفصلتين. وضع ملء الشاشة/سمة بديلة = تحسين CSS لا شاشة جديدة.
- **شاشة عرض العميل** (POSADVS_SCND) مبنية فعلاً (`/customer-display`).
- أي حاجة مناولة مستودعية تقع ضمن مسارات المخزون المبنية (POST028/029) أو ADR-006.

## العواقب

- (+) واجهة بيع واحدة تُصان بدل 2–3 نسخ Forms متباينة.
- (+) إغلاق شاشتين من قائمة النواقص بلا فقد وظيفة.
- (−) لا شيء جوهري — الوظائف مغطّاة أو بلا قيمة.
