# ADR-011 — استيراد Excel للكميات + طباعة باركود (POS_IMPXLS_AVLQTY): مؤجّل قابل للبناء

- **الحالة:** مقبول (⏳ مؤجّل — **ليس N/A نهائياً**)
- **التاريخ:** 2026-07-06
- **الشاشة:** POS_IMPXLS_AVLQTY (خدمي/Utility، 300 KB)

## السياق

الشاشة تستورد كميات/أصناف من ملف Excel وتطبع باركود لها.

- **proof:** `docs/screens/POS_IMPXLS_AVLQTY.md §1` → "أكثر الجداول: `IAS_PRINT_BARCODE_TMP`, `IAS_ITM_MST`, `IAS_PRICING_LEVELS`, `WAREHOUSE_DETAILS`" — استيراد بيانات صنف/سعر/كمية + جدول مؤقّت لطباعة الباركود.
- خلافاً لشاشات WMS/JV/Install، **لهذه الشاشة قيمة تشغيلية حقيقية** (تحميل دفعي للأصناف/الكميات + طباعة ملصقات) قد يطلبها العميل.

## القرار

**لا تُغلق كـN/A.** تُصنّف **مؤجّلة قابلة للبناء** ضمن الموجة D. القرار: **نبنيها لاحقاً** عند الحاجة، لا نستبعدها معمارياً.

## البديل (حتى تُبنى) + مسار البناء

- **طباعة الباركود** مبنية بالفعل في الواجهة (`features/print` — JsBarcode EAN-13/Code-128 + ESC/POS) — النصف الثاني من الشاشة جاهز.
- **الاستيراد** هو الناقص: عند البناء يكون:
  - `POST /catalog/import` (multipart CSV/XLSX) → تحقّق صفوف (كود/سعر/كمية/مخزن) → كتابة overlay في MOTECH_POS (ITEMS_OVERLAY الموجود) لا في ERP.
  - واجهة رفع ملف + معاينة + تقرير أخطاء صف-بصف، ثم زر «طباعة باركود» للأصناف المستوردة (يعيد استخدام feature الطباعة القائمة).
- **بديل فوري بلا شاشة:** الاستيراد الدفعي عبر migration/seed أو سكربت CLI عند الترحيل الأولي.

## العواقب

- (+) لا نُغلق بابَ ميزة ذات قيمة بادّعاء N/A خاطئ.
- (+) نصف العمل (الطباعة) منجز؛ البناء المستقبلي محدود بالاستيراد فقط.
- (−) تبقى في قائمة «قابلة للبناء» (لا تُحسب ✅ ولا 📄) حتى تُنفَّذ — تصنيف صادق.
