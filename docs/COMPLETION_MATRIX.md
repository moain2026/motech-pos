# COMPLETION_MATRIX — خارطة اكتمال Motech POS مقابل Onyx Pro

> **الغرض:** خارطة ثغرات (Gap Matrix) دقيقة لكل شاشة/عملية في Onyx POS (80 شاشة + عمليات المسارات الـ9) مقابل ما هو **مبنيّ فعلياً** في Motech POS.
> **المنهج:** proof-not-assumption. الحالة أُثبتت بالفحص الفعلي للكود:
> - **Backend المبنيّ (7 modules):** `auth, bills, catalog, customers, reports, returns, shifts` — endpoints مؤكدة من `openapi.json` والـ controllers.
> - **Frontend المبنيّ (7 features / 7 صفحات):** `LoginPage, PosPage(+Cart/ItemGrid/SaleSummary/CustomerAttach), BillsPage, BillDetailPage, ReturnsPage, CustomersPage, ReportsPage` + `ShiftBar`.
> - قاعدة الحكم: الشاشة تُعتبر **✅ مبنية** فقط إذا كان لها **backend endpoint + frontend UI** فعليان. أي نقص = 🟡. لا شيء = ❌.
> **التاريخ:** 2026-07-01. المرجع: `docs/screens/INDEX.md`, `docs/flows/`, `docs/db/PACKAGES_ANALYSIS.md`, `docs/SCREENS_PRIORITY.md`.

## الـ Endpoints المبنية فعلياً (خط الأساس المؤكد)

```
auth      : POST /auth/login · POST /auth/refresh · GET /auth/me
shifts    : POST /shifts/open · POST /shifts/{id}/close · GET /shifts/current · GET /shifts/{id}/summary
bills      : POST /bills · POST /bills/{id}/payments · GET /bills · GET /bills/{billNo} · GET /bills/posted/{id} · GET /bills/summary/daily
catalog   : GET /items · GET /items/{code} · GET /items/barcode/{bc}
customers : GET /customers · GET /customers/{code} · GET /customers/{code}/points      (قراءة فقط)
returns   : POST /returns · GET /returns · GET /returns/{id}
reports   : GET /reports/daily · GET /reports/monthly · GET /reports/by-item · GET /reports/by-machine   (4 تقارير)
```

**قيود جوهرية في المبنيّ حالياً (مثبتة من الكود):**
- **الدفع:** زرّان فقط (نقد/بطاقة) بالمبلغ الكامل (`SaleSummary.tsx`: `amount: bill.netAmt`). **لا** دفع جزئي/متعدّد، **لا** عملات متعددة، **لا** كوبون/نقاط/شيك/آجل.
- **البيع:** خصم رأس + خصم سطر فقط. **لا** فواتير معلّقة (HUNG)، **لا** عروض (`GNR_QTN_PRM_PKG`)، **لا** وصفة طبية، **لا** بطاقات خصم.
- **العملاء/الأصناف:** قراءة فقط — لا إدخال/تعديل من الواجهة.
- **لا يوجد إطلاقاً:** مقبوضات/مصروفات، مزامنة/فوترة إلكترونية، قارئ أسعار مستقل، تصفية كاشيرات/فائض-عجز، عهدة مالية، جرد، تحويلات مخزنية، أوامر بيع، إعدادات نظام، صلاحيات، لوحة مفاتيح لمسية، أجهزة، نسخ احتياطية.

- **صارم (كامل فقط):** 4 / 80 = **5%**.
- **مرجّح (كامل=1، جزئي=0.5):** (4 + 17×0.5) / 80 = 12.5 / 80 = **~16%**.
- **بالوظيفة الأساسية (دورة البيع MVP):** دخول✅ + وردية🟡 + بيع🟡 + دفع🟡 + مرتجع✅ + تقرير يومي🟡 → **~55%** من MVP قائم، لكن مع ثغرات حرجة (معلّقة، دفع متعدد، مطابقة إقفال).

- **النسبة الحالية للاكتمال:** ~**5% صارم** / ~**16% مرجّح** من الـ80 شاشة — لكن **~55% من دورة البيع الأساسية (MVP)** قائمة هيكلياً.
