## 2. مشاكل حرجة مكتشفة (Critical Issues)

### 🔴 P0 — التجاوب على الأجهزة (Responsive)
- **PosPage.tsx (شاشة البيع الأهم) = breakpoint واحد فقط!** الكاشير قد يستخدم تابلت/موبايل — الشاشة قد تنكسر.
- **0 استخدام useMediaQuery** — لا تكيّف ديناميكي (تبديل تخطيط حسب حجم الشاشة).
- SettingsPage = 2 breakpoint فقط (صفحة كبيرة تحتاج تجاوب أكثر).
- ✅ viewport meta موجود صحيح.
- **الإجراء:** فحص فعلي بالمتصفح (375/768/1280px) + إصلاح PosPage للمس والتابلت (وكيل ui-audit).

### 🔴 P0 — تغطية E2E ناقصة
- 4 ملفات e2e فقط لنظام محاسبي حرج. شركة برمجية محترفة تتطلب تغطية e2e لكل مسار حرج (بيع كامل، مرتجع، إقفال وردية، تقارير).
- **الإجراء:** بناء suite E2E شامل (Playwright) يغطّي دورة الكاشير الكاملة عبر الواجهة.

### 🟡 P1 — من التدقيق السابق (قائمة)
- باغ expected-cash بين close/reconciliation (waveF يصلحه).
- الترقيم غير مطابق تماماً لـ GET_BILL_NO_PRC في Onyx.
- المزامنة + الفوترة الإلكترونية = محاكاة (ليست حقيقية).
- auth محلي JSON (ليس USER_R).
- صفحة الإعدادات تغطّي 14/179 إعداد (settings-complete-100 يصلحها).

## 5. الحكم الناقد الصادق

- ✅ الأساس ممتاز (جودة كود عالية)
- ⚠️ التجاوب يحتاج إصلاح (PosPage حرج)
- ❌ E2E ناقص · صفحة إعدادات ناقصة (قيد الإصلاح) · مزامنة محاكاة
- النسبة الصادقة: ~44% تغطية شاشات، ~95% دورة كاشير، جودة كود ~85%.
