# اختبار الطباعة الميداني — 2026-07-13 (عتاد حقيقي عند المالك)

> أول اختبار على طابعات فعلية (بيئة التطوير بلا عتاد). النتائج موثّقة كما وردت من المالك.

## ✅ نجح: Bixolon SPP-R310 عبر البلوتوث
- **الطابعة:** Bixolon SPP-R310 — جوال حرارية، بلوتوث كلاسيكي (SPP).
- **النتيجة:** ✅ **الطباعة اشتغلت عبر البلوتوث** — إثبات حي أن طبقة `PrintTransport` + raster العربي يعملان على عتاد حقيقي.
- **المسار:** اقتران الطابعة على مستوى نظام أندرويد → تظهر كمنفذ تسلسلي → `Web Serial` يفتحها. (Web Bluetooth غير مستخدم — BLE فقط ولا يرى SPP، مؤكّد من وثائق Chrome الرسمية التي شاركها المالك).

## ❌ فشل (متوقّع): Xprinter XP-S260M عبر USB على Windows
- **الطابعة:** Xprinter XP-S260M — 80mm حرارية، ESC/POS، منافذ **RS232 + USB + LAN**، درج نقود 24V.
- **النتيجة:** ❌ لم تطبع عبر USB.
- **السبب الجذري (معروف وموثّق):** WebUSB يفشل على Windows — درايفر `usbprint.sys` يستحوذ على الجهاز فيمنع المتصفح من المطالبة به. (ليس عيباً في النظام — قيد منصة معروف).

### الحلول لهذه الطابعة (3، مرتّبة):
1. **LAN / شبكة (الأفضل — الطابعة تدعمها):** الطباعة عبر TCP:9100 مباشرة. يتطلب **تطبيق سطح مكتب (Tauri/Rust)** أو وسيط محلي (المتصفح لا يدعم TCP الخام).
2. **USB عبر درايفر النظام (يعمل الآن):** تثبيت درايفر Xprinter على Windows → طابعة نظام → الطباعة عبر مسار «طباعة المتصفح» (`window.print`) الحالي.
3. **RS232 / USB-serial (Web Serial):** توصيل عبر منفذ تسلسلي → `Web Serial` يفتحها (نفس مسار البلوتوث الناجح).

## الخلاصة والخطوة القادمة
- ✅ **طبقة الطباعة تعمل فعلياً على عتاد حقيقي** (بلوتوث مثبت).
- 🔜 **وسيط سطح المكتب (Tauri/Rust):** يحل USB على Windows + الطباعة الشبكية (LAN) مباشرةً بلا درايفر — الخطوة الاحترافية القادمة، بنفس واجهة `PrintTransport` (يُضاف ناقل `tauri-cmd` بثلاث قنوات: serial + spooler النظام + TCP:9100).
- **قيد الجوال المؤكّد:** بلوتوث SPP يعمل عبر Web Serial على أندرويد Chrome الحديث؛ للضمان الكامل على كل أجهزة الجوال → تطبيق Android (Capacitor) بناقل SPP مباشر.
