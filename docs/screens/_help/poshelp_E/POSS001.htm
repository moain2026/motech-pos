   POS Parameters and Settings

Use: POS settings are a set of general parameters that configures and adapts the POS system according to the organization’s policies and activities. The user must be careful upon dealing with these parameters as they direct the system to achieve the requirements, policies and objectives of the organization’s management.

How to use the screen

After clicking on the Modify button and activating the suitable options, the screen is used as follows:

First: Parameters

-     No. of User No. Digits: Here the number of digits in the user’s number is determined. Note that this number appears as a part of the document sequence number if the activated option in the Invoices Sequence parameter is (With Machine Number and User Number).

-     No. of POS No. Digits: Here the number of digits in the POS number appears as a part of the document sequence number if the activated option in the Invoices Sequence parameter is (With Machine Number and User Number) or (With Machine Number Only).  

-     No. of Serial No. Digits: Here the number of digits in the serial number for the invoices and returns is determined. It is important to note that this number’s effect appears in the Sales Invoices field and in the Sales Returns field. The serial number refers to the length of the sales invoice number and the length of the sales return as shown in the following invoice number: 

Number (21) that is shown at the beginning of the invoice number refers to the year 2021. It is followed by the number (1), which refers to the server number, then, there are three numbers (001) that are basically for the machine number or the POS device number. After that, the number of the user (1), followed by the five numbers of the invoice sequence, which are (00025). It’s important to mention that this numbering is mandatory, and the number’s length can be changed using the previous fields only.

-     Setting Time Forward/Backward /Number of Hours/ for Period from/ To:  These fields are used to determine the period by which the time will be set backward (in hours). Note that this step takes place in the sales invoice, if there are shifts that work on the system after midnight but the organization wants to record them for the day before. Consequently, the selling movements that are made after midnight will be recorded in the previous day’s date. For example, if the number of the hours has been set as three, all the transactions until 3:00 am will be dealt with as a movement of the previous day. This configuration might be set for a specific period.   

-     Invoices Sequence: Here, the mechanism of the invoice sequence is determined through the following two options: (With Machine Number and User Number) or (With Machine Number Only). The latter option, is related to the previous two fields (Number of Digits in User Number) and (Number of Digits in POS Number). 

-     Items Pricing: This parameter is used to determine the type of pricing level, and it includes one of the following options:

?  General: It is one level for all the POS. Upon selecting this option, the (Pricing Levels) field appears so that the default pricing level for all POS can be selected, with the ability to select any other level according to the user’s privileges.

?  At the Level of Each POS: It means that each POS has its own pricing level. Upon selecting it, the Pricing Levels field disappears and appears in the Add POS Screen in which the required level is selected for each POS separately.

-     Pricing Levels: This field appears upon selecting the (General) option in the previous parameter. It is used to determine the default pricing level at the level of all POS, with the ability to select any other level in the invoices according to the user's privileges.

-     Use F12 Key: This parameter is used to determine how to use the F12 key in the Sales Invoices screen, and it includes two options:

?  Cash Payment: Upon selecting this parameter, the shortcut of the Direct Cash Payment screen appears by using (F12) in the sales invoice. Through this screen, the payment can be made either in cash or by visa, noting that the Multiple Payment screen appears by using F10.

?  Multiple Payment: Upon selecting this parameter, the Multiple Payment screen appears by clicking on (F12) in the sales invoice. This screen allows selecting the payment method: Cash, Credit Card, or Check. Then, the user can also use F10 to view the Cash Payment screen.

-     Return and Replacement Type: This parameter is used to determine the return and replacement mechanism for the sold goods. Meaning that the replacement or the return cannot be made if the determined period has been exceeded. This parameter includes the following two options: 

?  General: Upon selecting this option, two fields will appear in order to determine the replacement period and the return period in days for all the POS. 

?  At the Level of Each POS: Upon selecting this option, the return period, as well as, the replacement period will be determined at the level of each POS separately.

-     Show Alert to User after Suspending Invoice: It is used to determine the number of the pending invoices, then an alert saying that (Pending invoices found. Check them) appears to the user with the pending invoices if the determined limit is reached.  

-     Returned Amounts Limit after Selling: It is used to enter a limit for the customer’s returned amounts in the Payment screen (Remaining amount after recording the paid amount by the customer in the Paid Amount field). The limit that will be entered here is used to monitor the discrepancy between the required amount and the paid amount, since it must not be exceeded.

Hint: The limit will be determined based on the largest traded category. It is known by default that the customer’s remaining amount will not exceed the largest category, and it is used to reduce making any mistakes upon entering the amounts in the Payment screen. The system displays an alert message saying (The remaining amount must be less than or equal to the remaining amount limit) upon exceeding the limit entered in the POS Settings screen.

-     Round Discount at the Item Level: It is used to round the discount in the POS invoice in which the discount amount appears with fractions, so that the discount amount appears without fractions. It includes the following parameters (Not used/ To the Nearest/Integer/ To the Highest Integer/ To the Lowest Integer).  

-     Pending Invoices Limit: It determines the number of the pending sales invoices or the allowed limit for suspending the sales invoices that have been suspended by the cashier before saving. If this limit is reached, a message appears saying (Cannot exceed the maximum number of pending invoices).

-     Customer Number Length / Mobile Number Length: It determines the length of the customer number and mobile number. The length cannot be exceeded upon adding the customer in the Customers Data screen.

-     Quantity Maximum Limit: Upon entering a quantity in this field, the system does not allow exceeding this limit the Invoices Screen at the item level. It is used to prevent the user from entering a quantity by mistake.

-     Number of Days to Display Sales Invoices: It determines the number of days during which the cashier can review the sales invoices that they have sold. Therefore, if this day’s limit is exceeded, the cashier will not be able to display these invoices.

-     Invoice Amount Rounding Method: This parameter is used to round the invoice amount to the category that will be recorded in the designated place. The discrepancy will be calculated as a discount, and it includes the following options:

?  As a discount to the nearest category.

?  Fraction as a discount to the nearest category.

?  Lowest fraction as a discount and highest discount as s charge to the nearest category.

-     Use Items Search: This parameter controls the items search method in the POS sales invoice and, it includes the following options:

?  Unused: Upon selecting the (Unused) option, it will not be allowed to display the items list. So that, in order to enter the items in the POS invoice the item number or the item barcode will be written either manually or using a (Barcode Reader).

?  Use F9 Key to Display Items: Upon selecting it, the F9 key is used in the Item Number field to display the item search list to search for the required item.

?  Use F9 to Display the Items Advanced Options: Upon selecting it, the F9 key is used in the Item Number field to display the advanced search list to search for the required item.

-     Use Pending Invoices: This parameter is activated in order to allow suspending a POS invoice. Provided that the user is granted the privilege to allow suspension in the Transactions Privileges screen.

-     Allow Printing Pending Invoices: This parameter is used to allow the user to either print the pending invoices by using Shift+F8 keys or by using the Additional Tools list.

-     Pending Invoices Audit: This parameter is used to audit the pending invoices, in which the data of the transactions that have been made on the pending sales invoices are stored. Plus, a report including all the added or deleted information that have been made on the invoice can be generated and printed.

-     Print Invoice after Saving Directly: This parameter is activated in order to automatically print the invoice after saving. This facilitates and speeds up the invoice issuance.

-     Allow Printing Invoices before Saving: This parameter is activated in order to allow printing the invoice before saving, using the Ctrl + Q key or the invoice transactions list.

-     Allow Modifying the Invoice Currency and Return Currency: Activating it allows the user to modify the invoice currency and the sales return by using F9 in the Currency field. So that, the currencies list appears in order to select the invoice currency. If this parameter is not activated, the currency appears automatically without being able to modify it.

-     Auto Show Payment Amount in Payment Screen by Currency: Activating it allows displaying the payment amount automatically in the Multiple Payment screen according to each currency. Therefore, the (Cashier) would not need to calculate the exchange rate in the Onyx ERP system. In addition, the user can scan the amount and write another amount according to the amounts received from the customer in the currency and so on.

Explanation: If the invoice value is 50,000 and part of it has been paid in a certain currency while the remaining part has been paid in another currency. This means that the amount will be entered in the first currency, and the remaining amount will be entered in the other currency and will automatically appear based on the exchange rate determined in the currency configuration.

-     Use the Shift +F6 Delete Key: Activating it allows the system to grant the cashier the privilege to delete the items in the Sales Invoice screen. Each item can be separately deleted, using Shift+F6 without an alert message. It’s important for the cashier to be granted the (Allow Deleting Items from POS Invoice) privilege in the Users’ Privileges screen in the POS system, particularly in the Transactions Privileges tab. Otherwise this parameter will not have any effect on the POS invoice.

-     Allow Deleting More Than a Record after Changing User: This parameter is activated if one of the users (The Admin) has been granted the privilege to delete items. Therefore, upon changing the user, this admin can delete more than one item (Record) without the need to change the user again before adding or modifying another transaction.

-     Monitor Items Deletion after Deleting: This parameter is selected to monitor items’ deletion in the Sales Invoice screen. The user can display a report with the time, date, device, and user who deleted the item from the invoice. This report is one of the (Auditing Reports) and it appears among the POS system reports.

-     Open System More Than Once: Upon activating this parameter, the user can open the POS system more than once. If it is not activated, the system will not allow opening it and an alert message will appear.

-     Open Drawer upon Payment: Upon activating this parameter, the drawer will be opened directly upon paying the sales invoice, whether in cash or using the multiple payment methods. Therefore, the user can enter the received cash from the customer in the drawer or return the remaining amount. The drawer is linked to the concerned port (Com) in the computer from the Add POS in Inputs screen.

-     Gather Duplicate Items upon Printing the Official Form: Upon activating this parameter, the quantities of the duplicate item will be gathered in the POS invoice when the invoice is printed. If it is not activated, the quantities will not be gathered, and the invoice items will appear upon printing in the same order that has been used in entry, even if there are duplicates.

-     Show Available Quantity in Invoices Screens: Upon activating this parameter, the available quantity of the item appears and is displayed in the sales invoices.

-     Auto-Save Invoice: Upon activating this parameter, the invoice will be (Temporarily Saved) automatically without clicking the Save button. This feature is beneficial in the cases of sudden power outage or accidental logout from the system. Therefore, upon re-logging into the system, the invoice will be displayed as it was before the interruption so that the user can complete the rest of the invoice data.

-     Show Item Image: Upon activating this parameter, an image of the item will appear in the sales invoice (below the invoice) when selected.

Second: Advanced Options

-     Use Cash Payment for Returns and Replacement: This parameter is used to determine the method of dealing with the returned goods from customers. It is also beneficial in determining how the returned goods will be processed in the system, and what financial and warehouse effects it will cause. This parameter includes the following options:

?  Pay upon Issuing Return Invoice: This option means that the selling transaction will be reversed, in which the goods will be returned and the customer will receive the return value. The return is done in the Onyx System sales. The accounting effect will be done once the return invoice is saved and posted to the Onyx Pro ERP system.

?  Cash Payment or Replacement: Using this option prevents the return from affecting the cash directly. It also allows replacing the items with other ones. The following are the options that this option provides for processing the return: 

?  Cash Payment for Return Value to Customer: If the return value is paid in cash, the Returns Cash Payment screen will be used after saving to issue the return value to customer. Afterwards, the posting will be done to the Onyx System using the Cash Payment Posting for Returns and Replacements screen.

Hint: As for the accounting effect of the return, the effect is credited on the Cash Payment and Replacements intermediary account that is added in the intermediary accounts or the returns cash intermediary account. This means that the effect is not credited on the cash account that is linked to the user, so that the accounting effect of the Returns Cash Payment screen is reversed on the cash account and the returns cash intermediary account in the cash account after posting the Cash Payment screen.

?  Replacement: If the goods are replaced with the return value, the return invoice number is used in the sales invoice upon paying. Moreover, the posting is done to the Onyx system by using the Post Cash Payment for Returns and Replacements screen. Note that the accounting and the warehouse effects vary in both cases.

-     Replacement Only: This option means that it is necessary to replace the goods only with the value of the returned goods from the customer. Note that the return value cannot be paid in cash to the customer.

   Remark: The Replacement intermediary account in the intermediary accounts has the priority over the specified account in the POS Settings and Parameters screen. Since it is at the level of each branch, and it allows recording the return value on the branches current account, which has been entered in the intermediary accounts, or the replacements and returns cash, if the return has been made in a branch, and the replacement or the payment has been made in another branch.

-     Returns and Replacements Intermediary Cash: This field appears upon selecting the (Cash Payment or Replacement) option in the previous parameter. It is used to determine the cash that has been added in the Cashes Data and linked to the replacements and returns account-which has been added in the charts of account, and it is the intermediary account in which the replacements and returns amounts are recorded-.

-     Returns and Replacements Intermediary Account: It is the number of the returns and replacements intermediary cash. It is fetched automatically after determining the cash in the previous field according the linking and configuration in the Cashes Configuration screen of the Onyx Pro ERP system.

-     Coupons Account: In this field, and upon using the pre-payment cards in the sales invoice payment, the coupons account that has been added in the chart of accounts is determined. The account is affected by the value of the coupons used in the payment transaction upon posting the coupons from the Post Cash Payment of Returns and Replacements screen.

-     Sales by Barcode Number Only: This parameter is used to determine whether the items will be entered in the POS invoice with the barcode number only or with the item number or barcode. It includes two options:

?  Unused: It allows entering the item number or the barcode number in the Item Number field in the POS invoice.

?  All Items: It prevents entering an item number in the Item Number field of the POS invoice. Therefore, entering the barcode number becomes mandatory.

-     Check Available Quantity: This parameter is used to determine whether the available quantity of the item will be checked or not in the invoice. It includes the following options (Allows, Not Allow, Allows with Alert).

-     Return Invoice Paid by Coupon: This parameter is used to determine whether an invoice that has been paid by a coupon will be returned or not. It includes the following options (Allows, Not Allow, Allows with Alert).

-     Employee Number Entering Method:  This parameter is used to determine whether the employee number will be entered in the POS invoice or not. It includes the following options:

?  Optional: Upon selecting this option, entering the employee's number in the invoice will be optional.

?  Mandatory: Upon selecting this option, entering the employee's number in the invoice will be mandatory, and the system will not allow saving the invoice unless it is entered.

?  By Commission Coding: Upon selecting this option, entering the employee's number in the invoice will be according to the commissions coding. If the invoice is linked to an employee's commission, entering the employee’s number will become mandatory. Otherwise, it will not be mandatory. 

-     Expiry Dates and Batches Using Method: This parameter is used to determine a method to deal with the expiry dates and the batch number in the system. It includes the following options:

?  Mandatory Enter Barcode: By selecting this option, it will not be allowed to enter the item number in the sales invoice. However, the system will require entering the barcode including the batch number and the expiry date. Therefore, entering the barcode becomes mandatory for the items that have a batch number or for the items that deal with expiry date.

?  Enter Barcode or Manually: In this case, the Expiry Date field and the Batch Number field will appear in the invoice. They can be read by entering the barcode, expiry date and batch number that are linked to the item. Note that the date and the batch can be entered manually or by using F9.

-     Use Customer Request: This parameter is used to determine whether the customer's request will be activated and used in the POS system or not. Moreover, it is also used to determine whether its use will be optional or mandatory or it will not be used at all. If the user selects (Optional or Mandatory), the Customer Request screen will appear among the system transactions.

-     Sales Clearances Entering Method: This option is used to determine whether the cashier sales will be cleared at the day level or at the period level. If the user selects (For One Day), the clearance will be at the day level (A Certain Date). While if the user selects (Multiple), the (From Date / To Date) and (From Time/To Time) fields will appear in the Clearance screen. Therefore, the clearance can be done for a certain period, or multiple clearances can be made in the same day. 

-     Type of Sales Return Approval: This parameter is used if the cashier does not have the privilege on the sales return while the sales returns must be approved by the sales supervisor or the authorized person. This parameter includes two options:

?  Upon Opening the Screen: It means that the cashier cannot access the Sales Return screen, while only the authorized person can access it. Therefore, the cashier will enter the return items, then saves the (Payment).

?  Upon Saving: It means that the cashier can access the return invoice and add the return items. However, the cashier will not be able to save the (Payment) as the sales supervisor or any authorized user, are the ones who have the privilege to approve the return.  

-     Sales Return Template: This option is used to select the (Screen Color) theme of the sales return invoice so that the user (Cashier) can distinguish it. 

-     Return Method of Promotional Offers Items: This option is used to determine the sales invoice return method if the sales invoice includes a promotional offer. Therefore, the user selects one of the following options: 

?  Return Items Optionally: If this option is selected, the returned items will be optionally fetched in the return invoice.

?  Return All Invoice Items: If this option is selected, all the returned items of the invoice will be mandatorily fetched in the return invoice. This includes all the items whether they have promotional offers or not. 

?  Return Invoice Items That Have Promotional Offers: If this option is selected, only the returned items that have promotional offers will be mandatorily fetched in the return invoice.  

-     Posting Method of Sales Invoice Issuance Note: This option is used to determine the posting method of the sales invoice issuance note to Onyx ERP System. It includes the following options:

?  Post Immediate Issuance Note: Upon selecting this option, the sales invoice will be immediately posted to the invoice issuance note.

?  Post Invoice Issuance Note by Available Quantity Only: Upon selecting this option, the sales invoices will be posted with a goods out note for the available quantities only.

?  Post Invoice without Invoice Issuance Note: Upon selecting this option, the sales invoice will be posted without the invoice issuance note.

-     Pay Invoices from Invoices Payment Screen: Upon activating this parameter, the (Invoices Cash Payment) screen appears in the system transactions. It is also used to separate the selling transaction from the payment transaction, noting that if this parameter is activated the system will not allow posting the invoices unless the payment is done. 

-     Not Allow Sales If Previous Sales Are Not Cleared: Upon activating this parameter, it becomes mandatory to use the Cashiers Sales Clearing screen. Moreover, the system will not allow making a selling transaction in the following day unless the previous day’s sales are cleared. Then, a message appears to the user saying that (Previous sales have not been cleared for this user).

-     Show Quantity in Item Deletion Screen: Upon activating this parameter, the item quantity appears in the (Item Deletion) screen in the invoice that appears upon clicking F3. Through this screen, the user can either determine the quantity that will be deleted or delete the whole quantity.

-     Use Customer Password in Credit Sales: Upon activating this parameter, the (Password) field appears in the sales invoices (Multiple Payment). Saving cannot be done without entering the password, noting that the password is entered in the customer data and is required from the customer upon selling; to monitor the credit sale transaction. 

Hint: Upon entering the customers data in the customers system, the customer’s password will be entered in the Password field.

-     Enter Mobile Number Mandatory: Upon activating this parameter, entering the customer’s mobile number in the invoice becomes mandatory upon payment.

-     Entering Customer Number in Cash Payment Is Mandatory: Upon activating this parameter, entering the customer’s number upon adding a cash invoice becomes mandatory. Meaning that the cash customers must be added in the system.

-     Show Customer Number Entry Screen upon Payment:  Upon activating this option, a screen appears for entering the customer mobile number or the customer number before payment transaction. This is considered a warning to the user to enter the mobile number and not to forget it, and if the mobile number is missing the Cash Customers Data screen is called in order to add the customer. This feature facilitates the user’s data entry instead of logging out the screen to complete the data entry in its designated place.

-     Add Cash Customers Data from Invoice Screen: Upon activating this option, the cash customers data can be added from the sales invoice. Therefore, once the mobile number is entered for the first time, the customer will be added to the cash customer data.

-     Use Customers Credit Limit in POS System: Upon activating this parameter, the system checks on the customer’s credit limit in the credit sales invoice. The system does not allow exceeding this limit unless the user has the privilege to do so.

-     Mandatory Enter Customer Card Number upon Credit Card Payment: Upon activating this option, entering the customer card number becomes mandatory in the Multiple Payment screen – if the payment method is by credit card-.

-     Mandatory Enter Reference Number upon Credit Card Payment:  Upon activating this option, entering the customer reference number becomes mandatory in the Multiple Payment screen – if the payment method is by credit card-.

-     Auto Show Credit Cards in Invoice: Upon activating this option, the credit cards appear automatically in the Multiple Payment screen to select the credit card with which the user wants to perform the transaction.

-     Use Prepaid Cards: Upon activating this parameter, the Coupon Number field appears in the Multiple Payment screen for the sales invoices of the prepaid cards. These coupons are used to pay the value of the invoice as they are already paid in advance to the organization by the authorized user who has activated the coupons system and distributed these coupons to the beneficiaries based on a receipt voucher.

-     Mandatory Enter Prepaid Cards Activation Period: This parameter can be activated if the (Use Prepaid Cards) parameter is activated. Upon activating this option, entering the activation period (From Date/To Date) becomes mandatory for each prepaid card. Note that entering the date for the prepaid cards can be carried out from the (Prepaid Cards and Coupons) screen in the system inputs.  

-     Use FOC Quantities: Upon activating this option, the FOC quantities can be used in the system. In the POS invoice, a column, designated for the FOC Quantities, will appear dedicated. Note that the settings of the FOC quantities are setup in the Onyx Pro ERP system.

-     Auto Fetch FOC Quantities from Promotional Offers: This option can be dealt after activating the (Use FOC Quantities) parameter. Activating this option allows fetching the FOC quantities automatically from the promotional offers, so that when the item number the FOC quantity that has been calculated from the promotional offer in the POS invoice will be read. In addition, the user can use the mouse’s right click to display the promotional offers.  

-     Collect Items Quantities with Promotional Offers by Quantity: Upon activating this parameter, the item's quantities will be collected in the POS invoice if there is a promotional offer “By quantity”. Therefore, the promotional offer can be applied.  

-     Show Cashiers Overage/Shortage Entry: Upon activating this parameter, the (Cashiers’ Sales Overage/Shortage) screen appears in the system transactions. Through this screen, the accounting entry of the cashiers’ sales overage/shortage can be created and displayed.

-     Allow Modifying Price: Upon activating this option, the item price can be modified in the POS invoice. While if it is not activated, the price modification will not be allowed in the invoice, also it will be required to grant the user the privilege to modify the price at the user level in the Users Privileges screen.

-     Allow Modifying Item Unit of Measurement: Upon activating this parameter, the system allows modifying the item’s unit of measurement in the POS invoice, if needed. While if it is not activated, the unit of measurement modification will not be allowed in the invoice.

-     Post Items Charges of Cash Invoices: This parameter is used in recording and posting the items’ charges of the cash invoices, *if there are any, upon posting them to the Onyx Pro system.

-     Use Prescription: Upon activating this option, the (Prescription) screen appears in the system transactions, which is used in the medical sector to perform the following:

?  Record the patient’s medications

?  Determine the dose

?  Determine the method of use

?  Determine the duration of use

?  Display all the other instructions recommended by the treating doctor

-     Show Item Total Amount in Net Value: Upon activating this parameter, the item’s net value will be displayed in the Total Amount column.

-     Use Discount Cards: This parameter shows a screen for coding discount cards, which is used in the Multiple Payment to obtain a discount for the invoice. After coding the discounts cards and defining them to specific customers, they are distributed according to the organization’s policy regarding the discounts. Then, after issuing the invoice to the customer, the discount card number is entered in the designated field to show the value of the discount directly based on the coding that has been done in the discount cards coding, noting that the customer will only pay the net amount of the invoice.

-     Method of Linking Discount Card to Customer: This parameter is used to determine the method of linking the discount card to the customer upon dealing with the discount card in the sales invoice. It includes two options:

?  Mandatory: The discount card must be linked to the customer in the Cash Customers screen. It becomes mandatory to enter the customer number in the sales invoice so that the discount card can be fetched.

?  Optional: The discount card is not linked to a customer, and it is not mandatory to enter the customer number in the sales invoice upon entering a discount card number.

-     Discount Cards Calculation Method: This parameter is used to determine the discount cards calculation method in the sales invoice. It includes the following options:

?  From Value of Items That Have No Offers or Discounts: The discount card will be calculated in the sales invoice from the items value that have no offers or discount. In another meaning, the items that have offers or discounts will be excluded upon calculating the discount card percentage.

?  From Value of Items That Have No Discount: The discount card will be calculated in the sales invoice from the items value that have no discount. In another meaning, that have offers or discounts will be excluded upon calculating the discount card percentage and the discount card will be calculated from the rest of the invoice items.

?  From Invoice Value: The discount card will be calculated in the sales invoice from the invoice value and without any exceptions for the offers and the discounts.

-     Not Show Net Sales Comparison with Cash Deposits: Upon activating this option, the Net Sales field and the Discrepancy field will disappear in the (Clear Cashiers’ Sales) screen. While if it is not activated, if will be possible to know the discrepancy between the net sales and the amounts received from the cashier upon clearing the cashiers’ sales.

-     Activate Representative Person Field: Upon activating this parameter, the user will be able to deal with the Representative Person Number field in the POS invoice.

-     Activate Employee Field: Upon activating this parameter, the user will be able to deal with the Employee Number field in the POS invoice.

-     Invoice Report Footer: Here, the user writes the text that they want to display at the bottom of the Sales Invoice screen with the ability to modify it at any time.

Third: Advanced Options:

-     Use Loyalty System: Ticking this parameter activates the loyalty system, which is the system that collects the customer’s points. Noting that setting up the loyalty system is done in the Customers System. 

-     Send Confirmation Message upon Redeeming Points: Upon activating this option, an SMS will be sent to the customer mobile (To Confirm) redeeming the points.

-     Customer Number Equals: Upon using the (Use Loyalty System), the customer number becomes either equal to the loyalty points card number or the mobile number. This happens in order to prevent duplicating the cards numbers or the mobile number. While if the loyalty system is not used, the customer number becomes equal to the mobile number only.

-     Loyalty Program Member Remark: The remark that appears to the Loyalty Program member in the sales invoice is written here. Therefore, the member can get to know more about the features of the loyalty program.

-     Remark of Loyalty Program Non-members: The remark that appears to the Loyalty Program non-members in the sales invoice is written here. Therefore, the non-members can get to know more about the features of the loyalty program, and consequently become members.

-     Sending Method: In this field, the method of sending the message to the customer is determined. The system provides several methods (SMS, Email, Telegram).

-     Send Message by: In this field, the recipient of the message will be determined. Meaning that, the user will determine whether the message will be sent to all customers, by the customers group, or selectively.

-     Message Text (Upon Opening Account/ Sales Invoice/ Sales Return/ for a Loyalty Program Member): In these fields, the message text that will be sent to the customer is coded according to each of the previously mentioned cases.

-     Invoice Report Footer: The text that will appear at the bottom of the sales invoice upon printing is coded here.

Fourth: Connection Type:

The engineers use this tab is used to determine the servers and the DB_LINK for each branch. It is also used to determine the connection data for the main center and the service name for each branch.

This tab is used as follows:

-     Branch Name: The branch name appears in this field once the branch number is entered.

-     Service Name: This field is used to determine the service of the branch through which the server will be linked to the Tns Name file. 

-     Databases Link: This field is used to specify the DB_Link of the connection between the sub-server and the main server.  The DB Link is determined for each branch, and it starts with the branch name, so that the branches can be distinguished from each other.

-     Description: This field is used to register the device name.

-     Server Number: This field is used in the serial numbering of the devices entered in the screen.
