 Sales Invoice 
 Use: This screen is used in the selling transactions in which 
the available quantity is not checked (Check Available Quantity parameter is not 
activated). The main purpose of using this screen is to make the selling 
transaction faster and easier for the cashier, without the need to wait for 
supplying the quantities of the sold items, noting that the warehouse effect 
takes place upon posting to the Onyx system. This screen is the core of POS 
system, as it allows making all the transactions that result in revenues, and 
granting discounts. Therefore, the organization can generate various types of 
reports on all the achieved revenues during a certain period. It is needless to 
say that this helps the organization in comparing the revenues of different 
periods. Plus, the sales invoices that are created using this screen are the 
only official documents that act as an obligation for both the organization and 
the items it sold, and the customers and the value of the invoice they have to 
pay. 
 This screen contains multiple main lists, each list contains a 
group of transactions. It is important to note that in front of each transaction 
there is a brief explanation. Below are the most important uses of the sales 
invoice: 
 - 
 Entering the 
sales transaction without checking the available quantity in order to without 
speed up the selling transaction. 
 - 
 Reducing the 
quantity of the items sold from the warehouse after posting to the Onyx Pro 
system. 
 - 
 Recording the 
financial effect after posting to the Onyx Pro 
system. 
 - 
 Entering the 
items by using a touch screen, and it must be activated from the system 
settings. 
 Several messages may appear upon opening the invoice, which means 
that there are necessary settings that must be made. Follow the below steps in 
order to hide these messages: 
 - 
 In order to 
prevent the (Enter Cash Account Number) message from popping up while using the 
screen, the (Cashier) user must be linked to a Cash number in the Users Data 
screen in the POS tab. 
 - 
 In order to 
prevent the (Enter Default Sale Currency) message from popping up while using 
the screen, the default sale currency must be determined. It is determined from 
the POS Machines and Devices screen at the level of each added 
POS. 
 How to Use Screen 
 Once the user 
views the invoice, it appears in Add status. It is needless to mention that this 
is one of the most important features of the POS sales invoice. While if the 
user selects Undo, the addition can be made using F11 from the keypad or by 
selecting the Add option from the dropdown menus. Below are the methods of 
working on a POS sales invoice: First: 
Master Data 
 - 
 Invoice 
Number: The invoice 
number appears automatically, and it is affected by the Sales Invoices Sequence 
Parameter in the (POS Settings) screen and the (Add POS) screen. The following 
is an explanation of the invoice number: 
 - 
 The invoice 
number starts with the year (21), followed by the server number (1), and the 
point number (001), and the user number (1), and lastly the invoice serial 
number (00001). Therefore, the number will be 
(211001100001). 
 - 
 Invoice 
Date: The invoice 
date appears automatically based on the date of the server or the sub-server and 
cannot be modified. 
 - 
 Time: The invoice 
time appears automatically as the time of the device is 
read. 
 - 
 Remarks: In this field, 
any remarks on the invoice are entered. 
 - 
 Employee 
Number: This field can 
be dealt with upon activating the (Activate Employee Field) parameter. Entering 
the employee number becomes mandatory upon selecting the (Mandatory or By 
Commission Coding) in the Employee Number Entry Method parameter. The employee 
is selected from among the employees who were added in the Employees Data screen 
in the Onyx Pro ERP system. 
 - 
 Representative 
Person Number: This field can 
be dealt with upon activating the (Activate Representative Person Field) 
parameter. It is used to enter the number of the sales representative person who 
performed the selling transaction in order to calculate the commission. Note 
that the representative persons can be added in the Sales Representative Persons 
Data screen in the Onyx Pro system. 
 - 
 Currency/ 
Exchange Rate: The default 
selling currency that has been selected in the POS Machines and Devices screen 
appears automatically, with the ability to select another currency upon 
activating the (Allow Modifying Invoice Currency and Return Currency) parameter 
in the POS Parameters and Settings screen. Moreover, the exchange rate that has 
been determined in the Currencies Configuration appears according to the 
selected currency or according to the selected user. 
 - 
 Other 
Data: These are 
additional fields that can be used to record the additional data of the 
invoice. 
 Second: Detailed Data 
 - 
 Item 
Number: Here, the 
number of the sold item is entered. Note that there are multiple methods to 
enter the item number in order to boost the cashier’s selling process 
efficiency, as well as, reduce the delay in serving customers. These methods are 
explained as follows: 
 ? 
 Item 
Barcode Reading Via Scanner: This method is 
considered the fastest method, as well as, the most commonly used in the items’ 
entry. Reading the item barcode becomes mandatory upon activating the (Sell with 
Barcode Number only) for all the items in the POS Parameters and Settings 
screen. 
 ? 
 Manual 
Entry of Item Number or Barcode Number: This method is 
one of the least used methods in the items’ entry, yet the user may need it at 
certain times. 
 ? 
 Use 
Keypad Shortcuts: This is one of 
the most widely used methods. The user can use (F9) upon selecting (Use F9 to 
Display Items) option in the POS Parameters and Settings screen. The user can 
also use (F8) or Ctrl+F to search for any item, whether by name or number, 
noting that the user can use any of the other keys that have been explained in 
the Help Keys screen to search for items. Moreover, the user can display the 
Advanced Search screen by activating the (Use Items Search) parameter and 
selecting the (Use F9 to Display Items as Advanced Options). It is important to 
note that upon selecting the search method as (Starts with) in the sales invoice 
and entering the first digits of the number, a list of items starting with these 
digits will appear. 
 ? 
 Additional 
Keypad: This method is 
one of the most important methods used to search for items in which an 
additional keypad is setup in the POS Inputs and linked to the required items to 
be used in fetching the items to the sales invoice. The additional keypad can be 
used by either using Ctrl + K, or selecting the (Additional Tools) from the 
dropdown menu (Additional Tools). 
 - 
 Item 
Name: The item name 
appears automatically upon entering the item number in any of the previous 
methods. 
 - 
 Item 
Unit: Upon entering 
Item Number, the item unit appears automatically, noting that it might be the 
item’s main unit or the sales unit. The user can select another unit by 
activating the (Allow Modifying Item Unit) in the POS Parameters and Settings 
screen, and then pressing F9 in the Item Unit field, or by selecting the 
(Display Item Units) from the drop-down menu (Invoice Transactions). It is 
important to note that the previous steps can only be done if the flag exists in 
the item’s record. 
 - 
 Quantity: Here, the (1) 
quantity appears, which is the default quantity of the item and it can be 
modified upon granting the user the (Allow Modifying Quantity in POS Invoice) 
privilege in the Users' Privileges. Moreover, the quantity can be modified 
either by using (F8) key or by selecting the (Modify Item Quantity) from the 
drop-down menu (Invoice Transactions). 
 - 
 FOC 
Quantity: This 
abbreviation stands for the FOC quantity and it appears upon activating the (Use 
FOC Quantities) in the POS Parameters and Settings screen. The FOC quantity 
appears automatically according to the FOC quantities settings that are made in 
the Onyx Pro ERP system, or it is automatically fetched from the promotional 
offers. 
 - 
 Price: Here, the item 
price appears automatically and can be modified if the (Manual Prices Entry) 
privilege is activated in the Transactions Privileges in the Onyx Pro ERP 
system. In the POS system, the price can be modified by activating the (Allow 
Modifying Price) parameter in the POS Settings and Parameters screen. Moreover, 
the price can be modified at the user level in the Users’ Privileges by moving 
to the Price Field using the drop-down menu (Invoice Transactions) or by using 
the Ctrl + P key. F9 can also be used to display a list of prices entered in the 
Pricing screen in order to select the required price upon granting the user this 
privilege. 
 - 
 Discount 
%: This field is 
used upon activating the (Use Discount at Item Level) parameter in the Customers 
System Parameters. In this field, the discount amount or percentage appears; 
therefore, the customers’ discounts can be set in the Customers’ System. Note 
that the discounts can be set through one of these screens: the Discount Coding 
Screen, Promotional Offers Screen, or Discount Cards Screen. While if the 
parameter is not activated, the discount can be manually entered, noting that it 
will be subject to the percentages that have been determined for the user 
according to the granted privileges. The user can use Ctrl+D to move to the 
Discount Field. 
 - 
 Tax: The data of 
this field appears automatically, representing the determined tax amount or 
percentage of the item based on the tax settings that have been made in the Onyx 
Pro ERP system. 
 - 
 Total: This field is 
used to display the total value of the item, which is the (Price - Items 
Discount + Tax) if the Display Item Final Price in Net parameter is activated. 
 - 
 Barcode 
Number: The system 
automatically displays the item's barcode number once the item barcode is read 
or entered in any of the previously explained methods. 
 Third: Invoice Totals 
 - 
 Previous 
Amount: This field 
displays the total amount that must be received from the 
customer 
 - 
 Remaining 
Amount: This field 
displays the remaining amount of the customer after 
payment. 
 - 
 Invoice 
Discount Percentage: This field is 
used to enter the discount percentage at the invoice level. Once the percentage 
is entered, the discount value appears in the Invoice Discount field, noting 
that the entered percentage must not exceed the percentage that has been 
determined for the (Cashier) user in the Transactions Privileges in Onyx System. 
This percentage has a priority over the percentage determined in the Users' 
Privileges in the POS system. The use can move to the Discount field, using the 
F6 key in order to add the discount 
percentage. 
 - 
 Invoice 
Discount: This field is 
used to enter the discount value at the invoice level. Once the value is 
entered, the discount percentage appears in the Discount Percentage field, 
noting that the discount value can be entered so that the system calculates the 
discount percentage or the discount percentage can be entered so that the system 
calculates the discount value. It is important to mention that the discount 
percentage must not exceed the percentage that has been determined for the 
(Cashier) user in the Privileges. The user can move to the Discount Field, using 
the F5 key to enter the discount value. 
 - 
 Total 
Quantity: It is used to 
display the total number of pieces (quantity) in the 
invoice. 
 - 
 Items 
Discount: This field 
displays the total discounts that have been calculated at each item 
level. 
 - 
 Discount 
after Tax: This field 
displays the total discount after calculating the 
tax. 
 - 
 Sales 
Tax: This field 
displays the total sales tax charged on the 
invoice. 
 - 
 Total: This field 
displays the total value of the invoice, which is the resulted from multiplying 
the Quantity Column by the Price Column. 
 - 
 Net 
Discount: This field 
displays the net discount, which is the sum of Invoice Discount field and Items 
Discount field. 
 - 
 Net: This field 
displays the invoice net amount automatically, after deducting the discounts and 
charges, and after adding the tax (if any). 
 Fourth: Search for Weighted Items and 
Lengths 
 The system provides a special way to search for the weighted items 
in the sales invoice, which is pressing Ctrl+W. Consequently, a screen appears 
through which the user can search for the required item using the weighted 
items' prefix that has been determined in the POS Settings screen or by item 
number or item name. Moreover, the user can use the drop-down menu (Additional 
Tools) in order to move to the same screen. While in order to search for the 
lengths, the user can move to the Search screen, using Ctrl+E or using the 
drop-down menu (Additional Tools) in order to move to the same screen. 
 How 
to Use Screen 
 - 
 Search 
Field: In this field, 
the user writes the item name or a part of it. Then, the user clicks on the Tap 
button in order to filter the results according to the specified name, noting 
that the search criterion must be (By Item Name). 
 - 
 Search 
Criterion: The search 
criterion is determined by the item number or item name in order to record the 
search data in the previous field. 
 - 
 Sort 
by: In this field, 
the method of sorting the items in the screen is determined. 
 Remark: 
 The 
items that appear in this screen are the items that have been selected as 
weighted items in the Items Data screen. 
 Fifth: Payment 
 The Payment screen is used to pay the invoice amount. The system 
provides two screens through which the payment can be made, namely the (Normal 
Payment screen and Multiple Payment screen). Below are the steps of dealing with 
both screens: 
 - 
 Normal 
Payment: This screen is 
used in the invoice payment, using the selected invoice currency without 
allowing the user to modify the invoice type or its currency or paying the 
invoice using more than one currency. The Payment screen can be accessed by 
selecting the Display Payment Screen from the drop-down menu (Invoice Currency. 
Moreover, it can be accessed using the F10 key or the F12 key, according to the 
selected method in the POS Parameters and Settings screen. This screen consists 
of several fields that will be explained as 
follows: 
 - 
 Required 
Amount: It is the 
amount required from the customer, noting that it is the net amount of the 
invoice after the deducting the discount 
value. 
 - 
 Paid 
Amount: It is the 
amount paid by the customer in cash or online. The invoice amount can be 
displayed directly in this field upon using the (Automatically Show Payment 
Amount in Payment Screen) parameter from the POS Settings 
screen. 
 Remark : 
In order to activate the online payment amount, the default visa type must be 
selected at the POS level. Upon entering the online payment amount, the payment 
will be deducted from the default visa type that has been defined at the point 
level. 
 - 
 Remaining 
Amount: It is the amount 
remaining for the customer after deducting the invoice value from the paid 
amount. The system prevents the remaining amount from exceeding the amount that 
has been determined in (Returns Amounts Limit after Sale) field in the POS 
Settings screen. If any attempts were made to exceed this limit a message will 
pop up, saying (The remaining amount must be less than or equal to the remaining 
amount limit). 
 - 
 Printing 
Form: This field is 
used in selecting the printing form, which is the minimized form –the default 
form-, or the official invoice form (A4).

 Sixth: Monitor 
Deletion 
 In order to monitor the 
deleted items, the user can use the designated parameter in the POS Parameters 
and Settings screen. The cashier may not be granted the privilege to delete 
items, while the system administrator has this privilege. Interestingly, the 
user can press on Ctrl+R buttons in the Invoice screen, and can also change the 
user from the (Change User) option at the bottom of the screen in the Tool Bar 
to facilitate logging in the system. Afterward, users will be asked to enter 
their username and password in the login screen that appears; therefore, the 
user will be able to delete the items, then the system will automatically return 
to the (Cashier) user. This process allows the user to perform all the actions 
that can be taken, using the username and password of the authorized user; so 
that, the user will be able to add or delete items, grant or cancel a discount, 
etc. 
 Seventh: Suspend Invoices 
 The invoices suspension 
is used in this screen if some customers do not complete shopping. The cashier 
sometimes has to suspend the invoice for the current customer and open an 
invoice for another customer. Note that the pending invoice can be retrieved 
anytime, once the customer decides to pay its value and receives its items. The 
cashier cancels the invoice’s suspension by pressing on Ctrl+3 or by selecting 
the (Retrieve Pending Invoices) from the toolbar at the bottom of the screen. 
While if the customer does not want to retrieve the invoice, it will be canceled 
by the system administrator. 
 In the sales invoices, an 
alert message appears to remind the user of the pending invoices, so that they 
can be processed and the suspension status ends. 
 Hint: The invoice suspension depends on what has been set in the 
POS Parameters and Settings screen, as well as the granted privileges such as 
allowing the invoices suspension and the allowed number of pending 
invoices. 
 Eighth: Toolbar 
 The toolbar contains a 
set of commands that are used in the system, and the following illustration will 
explain each of these commands. Each command has a shortcut in the keypad, which 
has been explained in the Help Keys screen. These commands can also be accessed 
from the drop-down menus above the screen. 
 - Open Invoice More Than Once: This command is used to allow opening the Sales Invoice 
screen more than once according to the privileges that have been granted to the 
user. Note that this command can be also carried out using Ctrl+B 
 - Undo: This command can be used upon activating the (Use Undo Key 
Ctrl+U) privilege in the Users Privileges, and it is used to undo the 
transaction that the user is performing in the screen they are working on. 
 - Add: This command is used to add a new invoice, and it appears 
as activated after the undo process. 
 - Delete Item: This command is used to delete an item from the invoice, 
and it can be carried out using the F3 key. 
 - Search for Item: This command is used to search for the item that will be 
sold, only if there is no barcode for the item. The user can also search for an 
item by using the (Ctrl+F) shortcut, or by pressing (F9), noting that these 
methods are only used if the designated parameter is activated in the POS 
Settings. 
 - Enter Discount on Invoice - Amount: This command is clicked if the user wants to grant the 
customer a (Discount) on the amount at the invoice level. This command is 
executed using the F6 key. 
 - Enter Discount on Invoice - Percentage: This command is clicked if the user wants to grant the 
customer a discount as a (Percentage) at the invoice level. This command is 
executed using the F6 key. 
 - Show Payment Screen: This command is used to show the Normal Payment screen. 
 - Show Multi-Payment Methods Screen: This command is used to show the multi-payment screen. 
 - Suspend Invoice: The invoice suspension can be dealt with by activating the 
(Use Pending Invoices) privilege in the Users’ Privileges, and it is used to 
suspend the sales invoice for any reason and return to it later. 
 - Retrieve Pending Invoices: This command is used to display the pending invoices in 
the screen and end their suspension or cancel the invoice. 
 - View and Print invoices: This command is used to view the previous sales invoices 
in order to print them. 
 - Cash Payment for Returns: This command is used to go to the Cash Payment for Returns 
screen. 
 - Customers Data: This command is used to move to the POS Customers Data 
screen. 
 - Lock Screen: This command is used to lock the screen that the user is 
working on and prevent anyone from working on it until the user's password is 
entered. 
 - Change User: This command is used to change the user for any reason, 
such as clearing cashier sales, adding a discount, deleting an item, or 
approving a return. If the cashier does not have the privilege, this command 
will be used to allow another user with a more privileges to access the same 
screen. 
 - Logout: This command is used to logout the screen on which the 
user is working.
